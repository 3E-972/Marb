Trees

Marb can display a tree based on a QStandardItemModel using the same principle than QTreeView.
The root corresponds to the first item int the modèle (i.e. the item at 0,0).
Here, an example of QStandardItemModel well filled:

There is three types of trees:

Vertical tree

In a vertical tree, the root is displayed at the top of the view. The children are displayed below their parent.

A vertical tree

This snippet creates an example of a vertical tree:


Horizontal tree

The structure is similar to a vertical tree. But, the item children are displayed in column to the right of their parent.

A horizontal tree

Radial tree

A radial tree displays items by placing the root in the center and arranging the items in circles around it. Each circle represents the depth of the nodes.
This view has a behavior a bit different from other views. It has therefore not quite the same functionality (it is not possible to change the type of connectors, for example).

A radial tree

void Tree::setSpacing( int horizontal, int vertical )

Set the horizontal and vertical spacings between the items.

void Tree::setConnectionPen( QPen pen )

Sets the QPen used to paint connections between items.

QPen Tree::connectionPen()

Returns the QPen used to paint connections between items.

QPointF Tree::scan( QModelIndex index, QPointF depthAndLeft )

This methods is used to scan a tree which its root is index and calculates the depth and the number of items at the the left of each item in the tree.
depthAndLeft defines the default values for the depth and the left position.
Returns the depth and the larger (i.e., the number of leafs) of the given tree.

void Tree::positionsInTree()

Determines the absolute position in the tree of each item.

void Tree::positionsInView()

Calculates the position of each item in the view. Uses the tree positions to determinate the view position.

QRect Tree::itemRect( index )

Returns a QRect defining the bounding rect int the view of the given item.
